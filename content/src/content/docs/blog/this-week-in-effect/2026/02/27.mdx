Welcome back to This Week In Effect (**TWIE**) - your weekly update of the latest developments in the Effect community and ecosystem.

Effect is a powerful TypeScript library that helps developers build complex, synchronous, and asynchronous programs. One key feature that sets Effect apart is how it leverages structured concurrency to provide features such as async cancellation and safe resource management, making it easier to build robust, scalable, and efficient programs.

## Technology

### Effect v4 Beta Updates

The beta has been moving fast since launch. Here's a roundup of the most notable changes that landed this week in the `effect-smol` repository:

- New `Tx` modules and refactored transaction model — **Michael Arnaldi** merged a significant [refactor](https://github.com/Effect-TS/effect-smol/pull/1515) introducing dedicated Tx modules to streamline the transaction model, improving consistency and ergonomics across transactional workflows.
- Schema improvements — A series of quality-of-life additions landed this week: `SchemaParser.makeOption` / `Schema.makeOption` for constructing schema values as Option, new `OptionFromUndefinedOr` and `OptionFromNullishOr` schemas, and a fix for `Schema.encodeKeys` encoding non-remapped struct fields correctly.
- OpenAPI & HTTP fixes — Fixed multipart file upload schema in OpenAPI and ensured OpenAI JSON schemas are properly transformed for AI integrations.
- CLI bug fix — Resolved an issue with global flag handling in mixed CLI contexts, where setting context was incorrectly stripped.

You can follow the full changelog directly in the [effect-smol repository](https://github.com/Effect-TS/effect-smol).
