Welcome back to This Week In Effect (**TWIE**) - your weekly update of the latest developments in the Effect community and ecosystem.

Effect is a powerful TypeScript library that helps developers build complex, synchronous, and asynchronous programs. One key feature that sets Effect apart is how it leverages structured concurrency to provide features such as async cancellation and safe resource management, making it easier to build robust, scalable, and efficient programs.

## Technology

### Effect v4 Beta Updates

Here are the main changes that landed in `effect-smol`.

- **HttpApi overhaul**: Added a code generation module (`HttpApi.gen`), reworked `HttpApiClient` so `urlBuilder` uses schemas and client-shaped APIs, renamed the `withResponse` option to `responseMode`, added `disableCodecs` option to `HttpClientEndpoint` constructors, fixed security middleware cache reuse, and updated docs to reflect automatic codec coercion for `HttpApiEndpoint`. Also allowed middleware to accept error arrays and fixed the `HttpApi.Any` constraint for `HttpApiClient.urlBuilder`.
- **Schema improvements**: Collapsed same-type literal branches into a single enum array in JSON Schema output, fixed `compactEnums` to support multi-value enum branches, renamed `MakeOptions.disableValidation` to `disableChecks` (now also applies constructor defaults when set), and fixed `Struct` utility return types (e.g. `pick`) to preserve the simplified shape instead of exposing raw `Pick<T, K>` types.
- **AI & streaming**: Fixed missing finish parts in `LanguageModel` streaming, fixed tool output persistence in streamed chat history, fixed the OpenRouter client HTTP referer header name, and added `Stream.timeoutOrElse`.
- **PostgreSQL & runtime**: Added a dedicated PG connection for `LISTEN`, switched to `pg_notify` instead of raw `NOTIFY` in `PgClient`, cleaned up `sql-pg` constructors and layers, fixed `Unify.unify` for Effect unions, flushed devtools queued messages on teardown, and tracked `ManagedRuntime` fibers in a scope.
- **Bug fixes & internals**: Fixed `Schedule.fixed` immediate catch-up for long-running iterations, re-exported core `ServiceMap` references in `effect/References`, fixed `Queue.takeN` example, fixed metro bundler issues, allowed `ServiceMap.Key` to be covariant, and improved `Prompt.select`/`multiSelect` highlight text color.

You can follow the full changelog directly in the [effect-smol repository](https://github.com/Effect-TS/effect-smol).

## Community Highlights

- The best "skill" for Claude when working with Effect? Just clone the Effect repo, as Michael suggests.

- Building composable systems pays off: Dhravya Shah shares how [Supermemory](https://supermemory.ai/) was built with swappable layers from day one using Effect, making a fully local version trivial to spin up.

  <Tweet id="2035524585391849581" />

&nbsp;

- The [Effect Schema course](https://www.effective.software/courses/effect-schema) by Hemanta Kumar Sundaray got 2 new chapters: optional/nullable fields and Standard Schema Integration. Free and hands-on!

&nbsp;

- [effect-jetbrains-plugin](https://github.com/kriegcloud/effect-jetbrains-plugin) – A JetBrains IDE plugin (WebStorm & IntelliJ IDEA Ultimate) that integrates the `@effect/tsgo` language server, bringing LSP features like diagnostics, code actions, completions, hover info, inlay hints, and symbol navigation. Also includes a Runtime Dev Tools window with Clients, Metrics, and Tracer tabs.

&nbsp;

- [effect-analyzer](https://jagreehal.github.io/effect-analyzer/) – A static analysis tool for Effect TypeScript programs by Jag Reehal. Without running your code, it extracts structural elements and auto-generates flowcharts, railway diagrams, service maps, error flows, concurrency views, and more in Mermaid format. Also calculates complexity metrics and supports semantic diffing between two versions of an Effect program.

&nbsp;

- [audius-mcp-atris](https://github.com/glassBead-tc/audius-mcp-atris) – An MCP server built with Effect-TS that connects Claude AI to the Audius music platform. Uses two meta-tools ("search" and "execute") to compress the entire Audius API into ~1,000 tokens of context, running LLM-generated code in a sandboxed QuickJS WASM environment for security.

&nbsp;

- Dealing with time-based logic in TypeScript? Samuel Huber shares how `TestClock` from Effect makes time-based testing a breeze — perfect for cronjobs, scheduling, and time assumptions — all running the same code as production thanks to dependency injection.
